# SLTI 小狸花人格测试 - 技术规格文档

## 项目概述

仿照 ABTI 项目，创建一个纯前端单文件应用，实现小狸花人格测试。通过 30 道场景题，从 5 大模型、15 个维度分析用户行为模式，匹配 26 种小狸花人格类型。

## 五大模型与15个维度

| 模型 | 维度代码 | 维度名称 |
|------|----------|----------|
| 热爱驱力 (Passion) | P1, P2, P3 | 沉迷深度、深究度、拟人化程度 |
| 产出创作 (Create) | C1, C2, C3 | 创作冲动、创作形式、分享欲 |
| 消费行为 (Spend) | Sp1, Sp2, Sp3 | 氪金意愿、周边购欲、线下参与 |
| 圈内社交 (Social) | So1, So2, So3 | 战斗性、社群活跃度、从众性 |
| 同好身份 (Identity) | Id1, Id2, Id3 | 专一度、入圈标准、圈外公开度 |

## 技术实现

### 架构设计
参考 ABTI-main/index.html 的单文件架构:
- 所有 HTML/CSS/JS 在一个文件中
- 三个页面状态: intro(首页)、test(测试)、result(结果)
- 使用 CSS 类切换控制页面显示

### 问题设计
- 30道常规题目 (每个维度2题)
- 1道特殊门控题 (用于触发 EXIT 类型)
- 每题3个选项，分值 1/2/3
- 题目随机打乱顺序

### 计分规则
```javascript
// 每个维度的2题分数相加 (范围 2-6)
// 映射到 L/M/H 等级:
// 2-3 → L (Low)
// 4   → M (Medium)  
// 5-6 → H (High)
```

### 匹配算法
使用曼哈顿距离计算相似度:
```javascript
// 相似度 = max(0, round((1 - distance / 30) × 100))
// distance = sum(|用户维度值 - 类型维度值|)
// L=0, M=1, H=2
```

### 特殊类型触发
- EXIT: 门控题选择"不再喜欢秦彻"
- UNKNOWN: 所有类型最高相似度 < 60%
